Principios de HTML

HyperText Markup Language

El Lenguaje de Marcado de Hipertexto (HTML) es el código que se utiliza para estructurar y desplegar una página web y sus contenidos. Por ejemplo, sus contenidos podrían ser párrafos, una lista con viñetas, o imágenes y tablas de datos.

HTML no es un lenguaje de programación; es un lenguaje de marcado que define la estructura de tu contenido. HTML consiste en una serie de elementos que usarás para encerrar diferentes partes del contenido para que se vean o comporten de una determinada manera. Las etiquetas de encierre pueden hacer de una palabra o una imagen un hipervínculo a otro sitio, se pueden cambiar palabras a cursiva, agrandar o achicar la letra, etc

Fuente: Mozilla - HTML

Principios de CSS

Cascade Style Sheets

CSS es el lenguaje de estilos utilizado para describir la presentación de documentos HTML o XML (en-US) (incluyendo varios languages basados en XML como SVG, MathML o XHTML). CSS describe como debe ser renderizado el elemento estructurado en la pantalla, en papel, en el habla o en otros medios.

CSS es utilizado para diseñar y dar estilo a las páginas web, por ejemplo, alterando la fuente, color, tamaño y espaciado del contenido, dividirlo en múltiples columnas o agregar animaciones y otras características decorativas. Este módulo proporciona un suave comienzo hacia el dominio de CSS con los conceptos básicos acerca de su funcionamiento, la sintaxis y la manera en que puedes comenzar a utilizarlo para agregar estilos al HTML.

Fuente: Mozilla - CSS

Principios de SVG

Scalable Vector Graphics

Los gráficos vectoriales escalables (SVG) son un lenguaje de marcado basado en XML para describir gráficos vectoriales bidimensionales. Como tal, es un estándar web abierto, basado en texto, para describir imágenes que pueden representarse limpiamente en cualquier tamaño y están diseñadas específicamente para funcionar bien con otros estándares web, como CSS, DOM, JavaScript y SMIL. SVG es, esencialmente, para los gráficos lo que HTML es para el texto.

Las imágenes SVG y sus comportamientos relacionados se definen en archivos de texto XML, lo que significa que se pueden buscar, indexar, programar y comprimir. Además, esto significa que pueden crearse y editarse con cualquier editor de texto o con software de dibujo.

En comparación con los formatos clásicos de imagen en mapa de bits, como JPEG o PNG, las imágenes vectoriales en formato SVG pueden representarse a cualquier tamaño sin perder calidad y pueden localizarse fácilmente actualizando el texto que contienen, sin necesidad de un editor gráfico para ello. Con las bibliotecas adecuadas, los archivos SVG pueden incluso localizarse sobre la marcha.

Elementos Avanzados de SVG

<path d="...">

Elemento genérico para describir cualquier forma. Este elemento dá mucha mayor libertad al momento de trazar un camino con lineas y curvas. Su atributo más importante es d, el cual recibe un string describiendo el camino. Este camino se decribe a través de distintos comandos:

Comando Nombre Descripción
M, m Mover hacia mueve el 'puntero' hacia un punto determinado, sin dejar rastro
L, l, H, h, V, v Línea hacia pinta una línea desde el punto actual hasta el punto especificado. h y v corresponden a líneas horizontales y verticales y solo necesitan una cantidad de pixeles a moverse
C, c, S, s Curva Cúbica de Bézier Las Curvas Cúbicas de Bezier son descritas por 4 puntos
Q, q, T, t Curva Cuadrática de Bézier Las Curvas Cuadráticas de Bezier son descritas por 3 puntos
A, a Arco Elíptico Se definen como pa porción de un elipse
Z, z Cerrar Camino Unirá el último punto del camino, con el punto inicial del mismo. En caso de no estas, queda una figura abierta

Podrás notar que las mismas letras se repiten en mayúscula y minúscula. Esto es porque la descripción del camino es case-sensitive, es decir, discrimina si la letra es mayúscula o minúscula, y su comportamiento difiere levemente:

  • Las letras mayúsculas reciben una coordenada absoluta donde moverse (x, y)
  • Las letras minúsculas reciben un cambio de posicion relativo a la posición actual en la que se encuentra (dx, dy)
